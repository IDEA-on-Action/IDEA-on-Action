# AdminNewsletter 관리 가이드

## 📋 개요

### 목적
**AdminNewsletter 페이지**는 웹사이트 뉴스레터 구독자를 관리하고 구독 통계를 확인하는 관리자 전용 페이지입니다. 뉴스레터 구독자의 이메일 주소, 구독 상태(대기/완료/취소), 구독일, 구독 경로 등을 조회하고, 구독자의 상태를 변경하거나 GDPR 규정에 따라 삭제할 수 있습니다.

### 주요 기능
✅ **구독자 목록 조회**: 이메일, 상태, 구독일, 구독 경로로 조회
✅ **통계 대시보드**: 전체 구독자, 확인 완료, 확인 대기, 구독 취소 현황
✅ **상태별 필터링**: 전체/확인 대기/확인 완료/구독 취소 구분
✅ **이메일 검색**: 실시간 부분 일치 검색
✅ **구독자 상태 변경**: pending → confirmed, confirmed → unsubscribed 등 상태 관리
✅ **구독자 삭제**: GDPR 준수하여 구독자 정보 영구 삭제
✅ **페이지네이션**: 50명씩 분할 표시
✅ **성장률 및 이탈률 추적**: 일일 신규 구독자, 구독 취소율 통계

### 접근 권한
- **필요 권한**: Admin 이상 (admin, super_admin 역할)
- **접근 경로**: `/admin/newsletter` (관리자 사이드바 → System → Newsletter)
- **권한 없을 경우**: 로그인 페이지로 리다이렉트

## 🎨 화면 구성

### 1️⃣ 목록 화면 (메인 페이지)

#### 상단 헤더
```
┌──────────────────────────────────────────┐
│ Newsletter Subscribers
│ 뉴스레터 구독자를 관리하고 통계를 확인하세요
└──────────────────────────────────────────┘
```

#### 📊 통계 카드 (4개)
- **전체 구독자**: 등록된 모든 구독자 수
  - 하단: 일일 성장률 (예: "+12 오늘")
  - 아이콘: 👥 (Users)

- **확인 완료**: 이메일 확인을 완료한 구독자 수
  - 하단: 전체 대비 비율 (예: "65.2%")
  - 아이콘: ✓ (CheckCircle, 초록색)

- **확인 대기**: 이메일 확인을 기다리는 구독자 수
  - 하단: "이메일 확인 필요"
  - 아이콘: ⏱ (Clock, 노란색)

- **구독 취소**: 구독을 취소한 구독자 수
  - 하단: 이탈률 (예: "8.3% 이탈률")
  - 아이콘: ✕ (XCircle, 회색)

#### 🔍 검색 및 필터 영역
```
┌─────────────────────────┐ ┌─────────────┐
│ 🔍 이메일 주소 검색...    │ │ 상태 선택   │
└─────────────────────────┘ └─────────────┘
```

**검색**: 구독자 이메일 주소로 실시간 검색 (부분 일치)
**상태 필터**:
- 전체 상태: 모든 구독자 표시 (기본값)
- 확인 대기: pending 상태만 표시
- 확인 완료: confirmed 상태만 표시
- 구독 취소: unsubscribed 상태만 표시

#### 📋 구독자 목록 테이블
| 컬럼 | 설명 |
|------|------|
| **이메일** | 구독자 이메일 주소 |
| **상태** | Badge로 표시 (확인 대기/확인 완료/구독 취소) |
| **구독일** | 구독 신청 날짜 (한국 날짜 형식, 예: "2025. 11. 20.") |
| **구독 경로** | 구독 위치 (Footer, Popup, Website, API) |
| **액션** | 상태 변경, 삭제 등 작업 (⋮ 메뉴) |

#### 📭 빈 상태
검색 결과나 필터링 후 구독자가 없으면:
```
📧 아이콘
"구독자가 없습니다" 또는 "검색 결과가 없습니다"
```

#### 페이지네이션
- 50명씩 분할 표시
- 예: "페이지 1 / 5 (전체 234명)"
- 이전/다음 버튼으로 이동
- 검색/필터 변경 시 자동으로 1페이지로 리셋

## 🔧 기능별 사용법

### 1. 통계 확인

#### 대시보드 보기
1. Newsletter 페이지에 접속합니다 (`/admin/newsletter`)
2. 상단 4개의 통계 카드를 확인합니다
3. 각 카드에서 다음 정보를 파악합니다:
   - **전체 구독자**: 현재 등록된 모든 구독자 수
   - **일일 성장률**: 어제 자정 이후 신규 구독자 수 ("+12 오늘" 형식)
   - **확인 완료**: 이메일 확인을 완료한 구독자 비율
   - **이탈률**: 구독 취소한 구독자 비율 (낮을수록 좋음)

#### 통계 해석
- **일일 성장률 "+12"**: 오늘 신규 구독 12명이 들어옴
- **확인 완료 65.2%**: 전체 구독자 중 65.2%만 이메일 확인 완료
- **이탈률 8.3%**: 전체 구독자 중 8.3%가 구독을 취소함

---

### 2. 구독자 목록 조회

#### 전체 구독자 조회
1. Newsletter 페이지에 접속합니다
2. 상태 필터를 "전체 상태"로 유지합니다 (기본값)
3. 최신순으로 모든 구독자가 표시됩니다
4. 테이블에서 이메일, 상태, 구독일, 구독 경로를 확인합니다

#### 다른 페이지 보기
1. 하단 페이지네이션에서 "다음" 버튼을 클릭합니다
2. 다음 50명의 구독자가 표시됩니다
3. "이전" 버튼으로 이전 페이지로 돌아갑니다

#### 날짜 순서 변경
현재 버전에서는 최신순(내림차순) 정렬만 지원합니다. 오래된순 정렬이 필요하면 개발자에게 요청하세요.

---

### 3. 상태별 필터링

#### 확인 대기 구독자 조회
1. 상태 필터 드롭다운을 클릭합니다
2. "확인 대기" 옵션을 선택합니다
3. **pending** 상태의 구독자만 표시됩니다
4. 이 구독자들은 이메일 확인이 완료되지 않은 상태입니다

**작업 흐름**: 확인 대기 상태의 구독자는 관리자가 "확인 완료로 변경" 또는 일정 기간(예: 7일) 후 자동으로 삭제할 수 있습니다.

#### 확인 완료 구독자 조회
1. 상태 필터에서 "확인 완료" 선택
2. **confirmed** 상태의 활성 구독자만 표시됩니다
3. 이 구독자들은 이메일로 구독을 확인한 상태입니다

#### 구독 취소 구독자 조회
1. 상태 필터에서 "구독 취소" 선택
2. **unsubscribed** 상태의 구독자만 표시됩니다
3. 이 구독자들은 구독을 취소한 상태입니다

#### 전체 상태 돌아가기
1. 상태 필터에서 "전체 상태" 선택
2. 모든 상태의 구독자가 다시 표시됩니다

---

### 4. 이메일로 구독자 검색

#### 검색 방법
1. 검색 입력창에 구독자의 이메일 주소를 입력합니다
2. 예: "user@example.com", "john", "@gmail.com"
3. 입력과 동시에 실시간으로 필터링됩니다

#### 검색 특징
- **부분 일치**: "user" 검색 시 "user@example.com", "superuser@email.com" 모두 표시
- **대소문자 구분 안 함**: "USER" 검색 = "user" 검색 (동일하게 동작)
- **실시간 필터링**: 글자마다 즉시 반영
- **페이지 리셋**: 검색 실행 시 자동으로 1페이지로 이동

#### 검색 예시
| 검색어 | 결과 |
|--------|------|
| "user@example.com" | 정확한 이메일 주소 검색 |
| "gmail" | "@gmail.com"으로 끝나는 모든 구독자 |
| "john" | 이메일에 "john"이 포함된 모든 구독자 |
| "2025-11" | 특정 패턴 검색 (되지 않음, 이메일 필드만 검색) |

#### 검색 초기화
1. 검색창의 텍스트를 모두 지웁니다
2. 전체 구독자가 다시 표시됩니다

---

### 5. 구독자 상태 변경

구독자의 상태를 변경하여 구독 프로세스를 관리합니다.

#### 5.1 Pending → Confirmed (확인 완료로 변경)

**사용 상황**: 구독자가 이메일 확인 링크를 클릭하지 않았지만, 이메일이 유효하다고 확인된 경우

**단계별 가이드**:
1. 상태 필터에서 "확인 대기"를 선택합니다
2. 상태를 변경할 구독자를 찾습니다
3. 구독자 행의 우측 끝에 있는 **⋮ (점 3개)** 버튼을 클릭합니다
4. 드롭다운 메뉴에서 **"확인 완료로 변경"** 옵션을 클릭합니다
5. 상태가 즉시 "확인 대기" → "확인 완료"로 변경됩니다
6. 하단에 토스트 알림이 표시됩니다: **"구독자 상태가 '확인 완료'로 변경되었습니다."**
7. 통계 카드가 자동으로 업데이트됩니다

**상태 변경 효과**:
- confirmed_at 필드에 변경 시간이 기록됩니다
- 이메일 발송 시 이 구독자가 대상이 됩니다

#### 5.2 Confirmed → Unsubscribed (구독 취소로 변경)

**사용 상황**: 구독자가 "구독 취소" 링크를 클릭했거나, 관리자가 구독을 취소해야 하는 경우

**단계별 가이드**:
1. 상태 필터를 "확인 완료"로 선택합니다
2. 구독을 취소할 구독자를 찾습니다
3. 구독자 행의 **⋮** 버튼을 클릭합니다
4. 드롭다운 메뉴에서 **"구독 취소로 변경"** 옵션을 클릭합니다
5. 상태가 "확인 완료" → "구독 취소"로 변경됩니다
6. 토스트 알림: **"구독자 상태가 '구독 취소'로 변경되었습니다."**

**상태 변경 효과**:
- unsubscribed_at 필드에 취소 시간이 기록됩니다
- 다음 이메일 발송 시 이 구독자는 제외됩니다

#### 5.3 Unsubscribed → Confirmed (구독 복구)

**사용 상황**: 구독을 취소했던 사용자가 다시 구독을 원하는 경우

**단계별 가이드**:
1. 상태 필터를 "구독 취소"로 선택합니다
2. 구독 복구할 구독자를 찾습니다
3. 구독자 행의 **⋮** 버튼을 클릭합니다
4. 드롭다운 메뉴에서 **"확인 완료로 복구"** 옵션을 클릭합니다
5. 상태가 "구독 취소" → "확인 완료"로 변경됩니다
6. 토스트 알림: **"구독자 상태가 '확인 완료'로 변경되었습니다."**

**상태 변경 효과**:
- confirmed_at 필드가 현재 시간으로 업데이트됩니다
- 다음 이메일 발송 시 이 구독자가 포함됩니다

### 6. 구독자 삭제 (GDPR 준수)

구독자 데이터를 완전히 삭제합니다. **이 작업은 되돌릴 수 없으므로 신중하게 진행하세요.**

#### GDPR이란?
GDPR (General Data Protection Regulation, 유럽 개인정보 보호법)은 사용자가 자신의 개인정보 삭제를 요청할 권리를 보장합니다. 사용자의 삭제 요청이 들어오면 즉시 처리해야 합니다.

#### 삭제 조건
- **관리자 권한**: admin 또는 super_admin 역할 필요
- **최종 확인**: 2단계 확인 (드롭다운 메뉴 + 확인 다이얼로그)
- **되돌릴 수 없음**: 삭제 후 복구 불가능

#### 단계별 삭제 가이드

**1단계: 구독자 찾기**
1. Newsletter 페이지에서 삭제할 구독자를 찾습니다
2. 검색 또는 필터를 사용하여 구독자를 찾을 수 있습니다
3. 구독자 이메일을 한번 더 확인합니다

**2단계: 삭제 옵션 선택**
1. 구독자 행의 우측 **⋮** 버튼을 클릭합니다
2. 드롭다운 메뉴 하단에 **"삭제 (GDPR)"** 옵션이 표시됩니다 (빨간 텍스트)
3. 이 옵션을 클릭합니다

**3단계: 확인 다이얼로그**
확인 다이얼로그가 나타나면:
```
┌──────────────────────────────────┐
│ 구독자 삭제 확인
├──────────────────────────────────┤
│ 이 작업은 되돌릴 수 없습니다.
│ 구독자 정보가 영구적으로 삭제됩니다.
│
│ GDPR 준수:
│ 사용자가 데이터 삭제를 요청한
│ 경우에만 이 기능을 사용하세요.
├──────────────────────────────────┤
│ [취소]        [삭제]
└──────────────────────────────────┘
```

**4단계: 최종 확인**
1. 메시지를 읽고 이 구독자를 정말 삭제할 것인지 확인합니다
2. "삭제" 버튼(빨간색)을 클릭합니다
3. 삭제가 즉시 실행됩니다

**5단계: 완료**
1. 토스트 알림이 표시됩니다: **"구독자가 삭제되었습니다."**
2. 목록에서 해당 구독자가 사라집니다
3. 통계 카드가 자동으로 업데이트됩니다

#### 삭제 실패 대응
만약 "구독자 삭제에 실패했습니다" 에러가 나타나면:
1. 관리자 권한을 확인합니다 (admin 이상이어야 함)
2. 네트워크 연결을 확인합니다
3. 페이지를 새로고침하고 다시 시도합니다
4. 계속 실패하면 개발자에게 문의합니다

### 7. 사용자 여정별 시나리오

#### 시나리오 1: 신규 구독자 이메일 확인 처리

**상황**: 어제 3명이 구독했지만 이메일 확인을 하지 않음

**처리 순서**:
1. Newsletter 페이지 접속
2. 상태 필터: "확인 대기" 선택
3. 확인 대기 중인 구독자 3명이 표시됨 (최신순)
4. 각 구독자를 확인하고 상태 변경:
   - 유효한 이메일: "확인 완료로 변경" 클릭
   - 스팸/유효하지 않은 이메일: "삭제" 선택
5. 완료

**참고**: 실제로는 자동 이메일로 확인 링크를 재발송하는 것이 좋습니다. 관리자가 수동으로 변경하지 않아야 합니다.

#### 시나리오 2: 구독자 요청으로 데이터 삭제

**상황**: 사용자가 "저의 구독자 정보를 삭제해주세요"라고 이메일로 요청함

**처리 순서**:
1. Newsletter 페이지에서 해당 사용자의 이메일로 검색
2. 구독자 찾기 (예: "john@example.com" 검색)
3. 구독자 행의 **⋮** 버튼 클릭
4. "삭제 (GDPR)" 선택
5. 확인 다이얼로그에서 "삭제" 클릭
6. 토스트 알림 확인 후 완료
7. 사용자에게 삭제 완료 회신 이메일 발송

**중요**: GDPR은 30일 내에 삭제 요청을 처리해야 합니다. 즉시 처리하는 것이 좋습니다.

#### 시나리오 3: 이메일 주소로 구독 취소 확인

**상황**: 특정 이메일의 구독 상태를 확인해야 함

**처리 순서**:
1. Newsletter 페이지 접속
2. 검색창에 이메일 입력 (예: "user@example.com")
3. 검색 결과 표시 (1명 또는 0명)
4. 상태, 구독일, 구독 경로 확인
5. 필요시 상태 변경 또는 삭제

## ❓ FAQ (자주 묻는 질문)

### Q1: 구독자를 수동으로 추가할 수 있나요?

**A**: 현재 버전에서는 수동 추가 기능이 없습니다. 구독자는 다음 방법으로만 추가됩니다:
- 웹사이트 Footer의 구독 폼
- 팝업 구독 다이얼로그
- API를 통한 프로그래밍 방식

수동 추가 기능이 필요하면 개발자에게 요청하세요. (향후 버전에 추가 예정)

---

### Q2: 구독자 데이터를 CSV 파일로 내보낼 수 있나요?

**A**: 현재 버전에서는 CSV Export 기능이 없습니다. 필요한 경우:
1. 개발자에게 CSV Export 기능 요청
2. 또는 Supabase Dashboard → SQL Editor에서 직접 내보내기

```sql
SELECT email, status, subscribed_at, metadata
FROM newsletter_subscriptions
ORDER BY subscribed_at DESC;
```

---

### Q3: Pending 상태가 7일 이상 지난 구독자는 어떻게 하나요?

**A**: 최적의 관행:
1. **자동 재발송**: 시스템이 3일째에 확인 이메일 재발송
2. **자동 삭제**: 7일 후 자동으로 pending 상태 구독자 삭제
3. **수동 처리** (현재):
   - 1주일마다 "확인 대기" 필터로 오래된 구독자 확인
   - 유효하지 않은 것으로 판단되면 삭제
   - 유효해 보이면 "확인 완료로 변경"

---

### Q4: 삭제한 구독자를 복구할 수 있나요?

**A**: **아니요, 삭제는 영구적입니다.** 복구 불가능합니다.

**이유**: GDPR 준수
- 사용자의 삭제 요청이 들어오면 즉시 처리해야 합니다
- 백업에서도 복구해야 삭제가 완료된 것으로 간주합니다

**따라서**:
- 삭제 전에 **반드시 신중하게 확인**하세요
- 실수로 삭제한 경우 개발자에게 즉시 알리세요 (긴급 복구 가능)

---

### Q5: 다수의 구독자를 한번에 삭제할 수 있나요?

**A**: 현재 버전에서는 개별 삭제만 지원합니다. 대량 삭제는:
1. **선택 삭제** (개발자 요청 가능):
   - 체크박스로 여러 개 선택
   - "일괄 삭제" 버튼으로 한번에 삭제

2. **현재**: 한 명씩 클릭하여 삭제

대량 삭제가 필요하면 개발자에게 요청하세요.

---

### Q6: 이메일 발송 시 구독 취소된 사용자는 제외되나요?

**A**: **예, 자동으로 제외됩니다.**

**이메일 발송 로직**:
- Status = "confirmed"인 구독자만 대상
- Status = "pending"인 구독자 제외 (미확인)
- Status = "unsubscribed"인 구독자 제외 (취소)

---

### Q7: 일일 성장률과 이탈률의 정확한 정의가 뭔가요?

**A**:

**일일 성장률** (Daily Growth)
- 정의: 지난 24시간 동안 신규 구독한 사용자 수
- 계산: 어제 자정 이후 subscribed_at을 가진 구독자 개수
- 표시: "+12 오늘" (예: 오늘 12명이 새로 구독)
- 참고: 모든 상태(pending, confirmed, unsubscribed)를 포함

**이탈률** (Churn Rate)
- 정의: 전체 구독자 대비 구독 취소한 구독자의 비율
- 계산: (unsubscribed 수 / total 수) × 100
- 표시: "8.3% 이탈률" 형식
- 해석: 낮을수록 좋음 (5% 이하 권장)

---

### Q8: 구독자 목록이 로딩되지 않으면 어떻게 하나요?

**A**:

**1단계: 기본 확인**
- 인터넷 연결 확인 (WiFi/모바일 데이터)
- 다른 웹사이트 접속 가능한지 확인

**2단계: 페이지 새로고침**
- F5 또는 Cmd+R (Mac) 키 누르기
- 또는 주소창에서 Enter 키

**3단계: 캐시 삭제**
- 브라우저 개발자 도구 (F12) → Network 탭
- "Disable cache" 체크박스 선택
- 페이지 새로고침

**4단계: 로그인 상태 확인**
- 로그아웃했는지 확인
- 다시 로그인 시도
- 관리자 권한 재확인

**5단계: 개발자에게 문의**
- 위 방법 모두 실패하면 개발자에게 보고
- 브라우저 콘솔(F12)에 나타나는 에러 메시지 공유

## 🔐 권한 및 보안

### 접근 권한
- **admin 역할**: 기본 권한
- **super_admin 역할**: 전체 권한 (다른 관리자 계정도 관리)
- **editor, contributor 역할**: 접근 불가 (403 Forbidden)
- **일반 사용자**: 접근 불가

### 권한 없을 때
```
"권한이 없습니다. 관리자 계정으로 로그인해주세요."
```
**해결**: super_admin에게 admin 권한 요청

### 보안 기능
✅ **Supabase RLS (Row Level Security)**: 데이터베이스 수준 보호
✅ **HTTPS 암호화**: 네트워크 통신 암호화
✅ **GDPR 준수**: 사용자 삭제 요청 즉시 처리
✅ **감사 로그**: 모든 구독자 상태 변경 기록

### 개인정보 보호
- 이메일 주소: 평문 저장 (필수)
- IP 주소: 익명화 처리
- User Agent: 저장 안 함 (향후 추가 가능)
- 삭제된 데이터: 복구 불가능

## 📞 트러블슈팅

### 문제 1: "권한이 없습니다" 에러가 나타남

**원인**: admin 또는 super_admin 권한이 없음

**해결 방법**:
1. 현재 로그인된 계정 확인
2. super_admin에게 admin 권한 부여 요청
3. 로그아웃 후 다시 로그인
4. 캐시 삭제 후 재시도

---

### 문제 2: 구독자 목록이 로딩되지 않음

**원인**: 네트워크 오류, DB 연결 실패, 또는 캐시 문제

**해결 방법**:
1. **네트워크 확인**
   - 인터넷 연결 상태 확인
   - 다른 웹사이트 접속 확인

2. **페이지 새로고침**
   - F5 또는 Cmd+R (Mac) 키
   - 또는 주소창의 새로고침 버튼

3. **캐시 삭제**
   - Ctrl+Shift+Delete (Windows) 또는 Cmd+Shift+Delete (Mac)
   - 캐시 및 쿠키 삭제

4. **다른 브라우저 시도**
   - Chrome, Firefox, Safari, Edge 중 하나로 시도

5. **개발자에게 보고**
   - 문제 지속 시 개발자에게 연락
   - 현재 사용 중인 OS, 브라우저 정보 제공

---

### 문제 3: 상태 변경이 실패함

**원인**: RLS 정책 또는 권한 문제, 네트워크 타임아웃

**해결 방법**:
1. **권한 재확인**
   - 관리자 권한 재확인 (admin 이상)

2. **로그인 다시 하기**
   - 로그아웃 후 재로그인
   - 세션 갱신

3. **페이지 새로고침**
   - F5 키로 페이지 새로고침
   - 다시 상태 변경 시도

4. **네트워크 확인**
   - 인터넷 연결 상태 확인
   - 다른 작업(검색, 필터링) 시도로 연결 확인

5. **개발자에게 보고**
   - 에러 메시지 스크린샷 제공
   - 어떤 상태로 변경했는지 설명
   - 어떤 에러 메시지가 나왔는지 공유

---

### 문제 4: 검색이 작동하지 않음

**원인**: 검색 문법 오류, 로딩 지연

**해결 방법**:
1. **검색어 확인**
   - 정확한 이메일 주소 입력했는지 확인
   - 띄어쓰기가 없는지 확인

2. **부분 일치 확인**
   - "user@example.com" 대신 "user" 또는 "example" 검색
   - 이메일의 일부로 검색

3. **필터 초기화**
   - 상태 필터를 "전체 상태"로 설정
   - 다시 검색 시도

4. **페이지 새로고침**
   - F5 키로 새로고침
   - 검색창 텍스트 모두 지우고 다시 입력
