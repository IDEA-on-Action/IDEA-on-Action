## 개요

**AdminNewsletter 페이지**는 뉴스레터 구독자를 관리하는 CMS 관리자 전용 페이지입니다. 구독자 목록 조회, 상태 변경, 통계 확인, GDPR 준수 삭제 등 뉴스레터 전반을 관리할 수 있습니다.

### 핵심 기능
- **통계 대시보드**: 전체/확인 완료/대기/취소 구독자 수, 일일 성장률, 이탈률 확인
- **검색 & 필터링**: 이메일 검색, 상태별 필터 (pending/confirmed/unsubscribed)
- **상태 관리**: pending → confirmed 승인, confirmed → unsubscribed 취소
- **GDPR 준수 삭제**: 2단계 확인 후 구독자 영구 삭제
- **CSV 내보내기**: 필터가 적용된 구독자 목록을 CSV 파일로 다운로드
- **페이지네이션**: 50개씩 표시, 대량 데이터 효율적 관리

### 사용자 권한
- **필요 권한**: `admin` 또는 `super_admin`
- **접근 제한**: 관리자 로그인 필수 (비로그인 시 `/login` 리다이렉트)
- **RLS 정책**: Supabase Row Level Security로 보호

## 주요 기능

### 1. 통계 대시보드 (4개 카드)
페이지 상단에 4개 통계 카드가 표시됩니다:

- **전체 구독자** (Users 아이콘)
  - 모든 상태의 구독자 총합
  - 일일 성장률 표시 (초록색 TrendingUp 아이콘, "오늘 +N명")

- **확인 완료** (CheckCircle 아이콘, 초록색)
  - 이메일 인증을 완료한 구독자 수
  - 전체 대비 비율 표시 (예: "85.3%")

- **확인 대기** (Clock 아이콘, 노란색)
  - 이메일 인증 대기 중인 구독자 수
  - "이메일 확인 필요" 메시지 표시

- **구독 취소** (XCircle 아이콘, 회색)
  - 구독을 취소한 사용자 수
  - 이탈률(churn_rate) 표시 (빨간색 TrendingDown 아이콘, "N.N%")

### 2. 구독자 목록 테이블 (5개 컬럼)
- **이메일**: 구독자 이메일 주소 (font-medium)
- **상태**: Badge 형식 (확인 대기/확인 완료/구독 취소)
- **구독일**: 한국어 날짜 포맷 (2025년 1월 15일)
- **구독 경로**: Footer, Popup, Website, API, 수동 추가
- **액션**: Dropdown 메뉴 (상태 변경, 삭제)

### 3. 검색 및 필터
- **검색창**: 이메일 주소 실시간 검색 (좌측 상단, Search 아이콘)
- **상태 필터**: 전체 상태, 확인 대기, 확인 완료, 구독 취소
- **조합 검색**: 검색어 + 상태 필터 동시 적용 가능

### 4. 페이지네이션
- **표시 개수**: 50개씩 (고정)
- **네비게이션**: 이전/다음 버튼 (ChevronLeft/ChevronRight 아이콘)
- **현재 페이지**: "페이지 1 / 5 (전체 234명)" 형식
- **자동 스크롤**: 페이지 변경 시 상단으로 스크롤

### 5. CSV 내보내기
- **버튼 위치**: 우측 상단 "CSV 내보내기" (Download 아이콘)
- **필터 적용**: 현재 검색/필터 조건이 적용된 데이터만 내보내기
- **파일명**: `newsletter-subscribers-YYYY-MM-DD.csv`
- **인코딩**: UTF-8 BOM (Excel 한글 깨짐 방지)

## 단계별 가이드

### 1. 통계 대시보드 확인하기

**목적**: 뉴스레터 구독자 현황을 한눈에 파악합니다.

#### 1.1 전체 구독자 수 확인
- **위치**: 좌측 상단 첫 번째 카드
- **표시 내용**:
  - 숫자: 전체 구독자 수 (1,234명 형식)
  - 성장률: 오늘 신규 구독자 수 (초록색, "+5 오늘")
- **해석**:
  - 성장률이 0이면 "변동 없음" 표시
  - 성장률이 양수면 초록색 TrendingUp 아이콘과 함께 표시

**예시**:
```
전체 구독자
1,234
+5 오늘 (초록색, TrendingUp 아이콘)
```

#### 1.2 확인 완료 비율 확인
- **위치**: 두 번째 카드 (초록색 CheckCircle 아이콘)
- **표시 내용**:
  - 숫자: 이메일 인증 완료 구독자 수
  - 비율: 전체 대비 비율 (85.3%)
- **목표**:
  - 건강한 뉴스레터: 70% 이상
  - 주의 필요: 50% 이하 (이메일 인증 프로세스 점검)

#### 1.3 확인 대기 관리
- **위치**: 세 번째 카드 (노란색 Clock 아이콘)
- **표시 내용**:
  - 숫자: pending 상태 구독자 수
  - 메시지: "이메일 확인 필요"
- **조치 사항**:
  - pending 상태가 7일 이상 유지되는 경우 재발송 고려
  - 대량 pending 발생 시 이메일 발송 시스템 점검

#### 1.4 이탈률 모니터링
- **위치**: 네 번째 카드 (회색 XCircle 아이콘)
- **표시 내용**:
  - 숫자: unsubscribed 상태 구독자 수
  - 이탈률: 전체 대비 비율 (2.5%, 빨간색 TrendingDown 아이콘)
- **이탈률 해석**:
  - **건강**: 5% 이하
  - **주의**: 5~10% (콘텐츠 품질 점검)
  - **위험**: 10% 이상 (긴급 개선 필요)

**스크린샷**: [AdminNewsletter - 통계 대시보드 4개 카드]

### 2. 구독자 검색하기

**목적**: 특정 이메일 주소를 가진 구독자를 빠르게 찾습니다.

#### 2.1 검색창 사용
1. 좌측 상단 검색창 클릭 (Search 아이콘)
2. 이메일 주소 또는 일부 입력
   - 전체: `user@example.com`
   - 부분: `user` (user로 시작하는 모든 이메일)
   - 도메인: `@gmail.com` (Gmail 사용자 모두)
3. 실시간 필터링 (타이핑 즉시 반영)
4. 검색 결과 확인 (하단 테이블)

#### 2.2 검색 범위
- **검색 대상**: 이메일 주소 (email 컬럼만)
- **대소문자**: 구분 없음 (`User@Example.com` = `user@example.com`)
- **부분 일치**: 지원 (`user` → `user@example.com`, `testuser@gmail.com`)

#### 2.3 검색 초기화
- 검색창 내용 삭제 (Clear 버튼 없음, 수동 삭제)
- 전체 목록 다시 표시

**검색 예시**:
```
검색어: "test"
결과:
- test@example.com
- testuser@gmail.com
- user@test.com
```

**팁**: 특정 도메인의 구독자를 찾으려면 `@domain.com`으로 검색하세요.

**스크린샷**: [AdminNewsletter - 이메일 검색]

### 3. 상태별 필터링

**목적**: 특정 상태의 구독자만 조회합니다.

#### 3.1 상태 필터 드롭다운
- **위치**: 검색창 우측
- **기본값**: "전체 상태"
- **옵션**:
  - 전체 상태 (all)
  - 확인 대기 (pending)
  - 확인 완료 (confirmed)
  - 구독 취소 (unsubscribed)

#### 3.2 필터 적용
1. 드롭다운 클릭
2. 원하는 상태 선택
3. 페이지 자동 리셋 (1페이지로 이동)
4. 필터링된 목록 표시

#### 3.3 조합 필터링
검색어와 상태 필터를 동시에 사용할 수 있습니다:

**예시 1**: 확인 대기 중인 Gmail 사용자
```
검색어: "@gmail.com"
상태: 확인 대기
결과: pending 상태의 Gmail 사용자만 표시
```

**예시 2**: 특정 사용자의 구독 취소 여부 확인
```
검색어: "user@example.com"
상태: 구독 취소
결과: 해당 사용자가 unsubscribed 상태인지 확인
```

#### 3.4 필터 초기화
- 드롭다운에서 **"전체 상태"** 선택
- 검색창 비우기

**스크린샷**: [AdminNewsletter - 상태 필터 드롭다운]

### 4. 구독자 상태 변경하기

**목적**: 구독자의 상태를 수동으로 변경합니다.

#### 4.1 상태 변경 Dropdown 메뉴
1. 목록 테이블 우측 **액션** 컬럼에서 **MoreVertical 아이콘** 클릭
2. Dropdown 메뉴 열림
3. 상태에 따라 사용 가능한 옵션 표시

#### 4.2 상태별 가능한 액션

**pending → confirmed (확인 완료로 변경)**
- **옵션 이름**: "확인 완료로 변경"
- **아이콘**: CheckCircle (초록색)
- **사용 시나리오**:
  - 사용자가 이메일 인증을 못 받았으나, 전화/채팅으로 본인 확인한 경우
  - 테스트 계정 수동 승인
  - 대량 가입 시 일부 계정 빠른 승인
- **결과**:
  - status: `pending` → `confirmed`
  - confirmed_at: 현재 시각으로 업데이트
  - Toast 알림: "구독자 상태가 '확인 완료'(으)로 변경되었습니다."

**confirmed → unsubscribed (구독 취소로 변경)**
- **옵션 이름**: "구독 취소로 변경"
- **아이콘**: XCircle (회색)
- **사용 시나리오**:
  - 사용자가 채팅/전화로 구독 취소 요청
  - 잘못된 이메일 주소 (반송 메일 많음)
  - 스팸 신고 받은 계정
- **결과**:
  - status: `confirmed` → `unsubscribed`
  - unsubscribed_at: 현재 시각으로 업데이트
  - Toast 알림: "구독자 상태가 '구독 취소'(으)로 변경되었습니다."

**unsubscribed → confirmed (확인 완료로 복구)**
- **옵션 이름**: "확인 완료로 복구"
- **아이콘**: CheckCircle (초록색)
- **사용 시나리오**:
  - 사용자가 재구독 요청
  - 실수로 취소한 경우
  - 테스트 후 복구
- **결과**:
  - status: `unsubscribed` → `confirmed`
  - confirmed_at: 현재 시각으로 업데이트
  - Toast 알림: "구독자 상태가 '확인 완료'(으)로 변경되었습니다."

#### 4.3 상태 변경 주의사항
- **되돌릴 수 있음**: 상태 변경은 다시 변경 가능 (삭제와 다름)
- **타임스탬프 업데이트**: 상태 변경 시 해당 날짜 자동 기록
- **권한 체크**: 관리자 권한 필수 (권한 없으면 에러 Toast)

**스크린샷**: [AdminNewsletter - 상태 변경 Dropdown 메뉴]

### 5. 구독자 삭제하기 (GDPR)

**목적**: GDPR 준수를 위해 사용자 요청 시 구독자 데이터를 영구 삭제합니다.

#### 5.1 삭제 절차
1. 액션 컬럼 Dropdown 메뉴 클릭 (MoreVertical 아이콘)
2. **"삭제 (GDPR)"** 옵션 선택 (빨간색, AlertTriangle 아이콘)
3. 확인 대화상자 표시

#### 5.2 확인 대화상자 (2단계 확인)
**제목**: "구독자 삭제 확인"

**설명**:
```
이 작업은 되돌릴 수 없습니다. 구독자 정보가 영구적으로 삭제됩니다.

GDPR 준수: 사용자가 데이터 삭제를 요청한 경우에만 이 기능을 사용하세요.
```

**버튼**:
- **취소**: 삭제 중단, 대화상자 닫기
- **삭제**: 영구 삭제 실행 (빨간색 버튼, bg-red-600)

#### 5.3 삭제 완료
1. **"삭제"** 버튼 클릭
2. Toast 알림: "구독자가 삭제되었습니다."
3. 목록에서 즉시 제거
4. 통계 대시보드 자동 업데이트

#### 5.4 GDPR 준수 가이드라인
**GDPR Article 17 (Right to Erasure, "잊혀질 권리")**

**삭제 가능한 경우**:
- ✅ 사용자가 명시적으로 삭제 요청
- ✅ 동의 철회 (구독 취소 후 데이터 삭제 요청)
- ✅ 데이터 보관 기간 만료 (법적 근거 소멸)
- ✅ 불법 처리된 개인정보

**삭제 불가능한 경우**:
- ❌ 법적 의무 준수 (세법, 상법 등)
- ❌ 공익 목적 (공중 보건, 역학 연구)
- ❌ 법적 주장 설정/행사/방어

#### 5.5 삭제 전 체크리스트
- [ ] 사용자가 실제로 삭제를 요청했나요?
- [ ] 법적 보관 의무가 없나요? (세법: 5년, 상법: 10년)
- [ ] 삭제 요청을 문서화했나요? (이메일, 티켓 번호)
- [ ] 삭제 전 백업이 필요한가요? (감사 목적)
- [ ] 비공개(unsubscribed) 상태로 두는 것이 더 나은 선택 아닌가요?

**스크린샷**: [AdminNewsletter - 삭제 확인 대화상자]

### 6. CSV 내보내기

**목적**: 구독자 목록을 Excel/Google Sheets에서 분석하기 위해 CSV 파일로 내보냅니다.

#### 6.1 CSV 내보내기 절차
1. 우측 상단 **"CSV 내보내기"** 버튼 클릭 (Download 아이콘)
2. 현재 검색/필터 조건이 적용된 데이터 내보내기
3. 파일 자동 다운로드
4. Toast 알림: "N명의 구독자 데이터를 내보냈습니다."

#### 6.2 파일명 및 형식
- **파일명**: `newsletter-subscribers-YYYY-MM-DD.csv`
  - 예: `newsletter-subscribers-2025-11-22.csv`
- **인코딩**: UTF-8 with BOM (Excel 한글 깨짐 방지)
- **구분자**: 쉼표 (`,`)
- **줄바꿈**: LF (Unix 스타일)

#### 6.3 CSV 컬럼 구조
```csv
Email,Status,Subscribed At,Confirmed At,Unsubscribed At,Source
user@example.com,confirmed,2025-01-15T12:34:56Z,2025-01-15T12:40:00Z,,footer
test@gmail.com,pending,2025-01-20T08:00:00Z,,,website
old@example.com,unsubscribed,2024-12-01T10:00:00Z,2024-12-01T10:05:00Z,2025-01-10T15:00:00Z,api
```

**컬럼 설명**:
1. **Email**: 구독자 이메일 주소
2. **Status**: pending/confirmed/unsubscribed
3. **Subscribed At**: 구독 신청일 (ISO 8601 형식)
4. **Confirmed At**: 이메일 확인일 (없으면 빈 문자열)
5. **Unsubscribed At**: 구독 취소일 (없으면 빈 문자열)
6. **Source**: 구독 경로 (footer/popup/website/api/manual)

#### 6.4 필터 적용 예시
**예시 1**: 확인 완료된 구독자만 내보내기
```

CSV 내보내기 클릭

```

**예시 3**: 전체 구독자 내보내기
```

CSV 내보내기 클릭

```

## 구독자 상태 이해하기

### 상태 전환 다이어그램
```
                  pending (확인 대기)
                      ↓
         (이메일 인증 또는 관리자 승인)
                      ↓
                 confirmed (확인 완료)
                      ↓
            (사용자 취소 또는 관리자 변경)
                      ↓
              unsubscribed (구독 취소)
                      ↓
          (관리자 복구 또는 사용자 재구독)
                      ↓
                 confirmed (확인 완료)
```

### 각 상태 설명

#### pending (확인 대기)
- **색상**: 노란색 Badge (bg-yellow-100)
- **아이콘**: Clock
- **의미**: 구독 신청은 했으나 이메일 인증 미완료
- **유지 기간**: 보통 24시간~7일
- **조치 사항**:
  - 7일 이상 pending 시 인증 이메일 재발송
  - 14일 이상 pending 시 자동 삭제 고려 (선택 사항)

#### confirmed (확인 완료)
- **색상**: 초록색 Badge (bg-green-100)
- **아이콘**: CheckCircle
- **의미**: 이메일 인증 완료, 뉴스레터 발송 대상
- **조치 사항**:
  - 정기 뉴스레터 발송
  - 이메일 반송률 모니터링

#### unsubscribed (구독 취소)
- **색상**: 회색 Badge (bg-gray-100)
- **아이콘**: XCircle
- **의미**: 구독 취소, 이메일 발송 제외
- **조치 사항**:
  - 이메일 발송 금지 (법적 의무)
  - 데이터는 유지 (재구독 가능)
  - GDPR 요청 시 삭제

## 구독 경로 (subscribed_via)

구독자가 어디서 구독했는지 추적하는 필드입니다.

### 구독 경로 종류

#### footer (Footer)
- **의미**: 웹사이트 Footer의 Newsletter 구독 폼
- **표시**: "Footer"
- **경로**: 모든 페이지 하단 Footer 컴포넌트

#### popup (Popup)
- **의미**: 팝업 또는 모달 구독 폼
- **표시**: "Popup"
- **경로**: 페이지 진입/이탈 시 표시되는 팝업

#### website (Website)
- **의미**: 웹사이트 메인 페이지 또는 전용 구독 페이지
- **표시**: "Website"
- **경로**: 홈페이지 Hero 섹션 또는 /newsletter 페이지

#### api (API)
- **의미**: 외부 API를 통한 구독
- **표시**: "API"
- **경로**: Zapier, Make, 또는 커스텀 통합

#### manual (수동 추가)
- **의미**: 관리자가 수동으로 추가
- **표시**: "수동 추가"
- **경로**: AdminNewsletter 페이지 (현재 미지원, 향후 추가 예정)

### 구독 경로 활용
**전환율 분석**:
```
Footer: 5% (100명 노출, 5명 구독)
Popup: 15% (50명 노출, 7.5명 구독)
Website: 20% (200명 방문, 40명 구독)
```

**마케팅 최적화**:
- Footer 전환율 낮음 → CTA 문구 개선
- Popup 전환율 높음 → 더 자주 표시
- Website 전환율 높음 → 전용 페이지 강화

## FAQ

### Q1: 구독자를 대량으로 추가할 수 있나요?
**A**: 현재 AdminNewsletter 페이지에서는 개별 추가만 지원합니다. 대량 추가는 다음 방법을 사용하세요:
- **CSV Import** (향후 기능): CSV 파일 업로드로 대량 추가 예정
- **API 사용**: Supabase API를 통해 프로그래밍 방식으로 추가

### Q2: 이메일을 잘못 삭제했어요, 복구 가능한가요?
**A**: 아니요, **삭제는 되돌릴 수 없습니다**. GDPR 준수를 위해 삭제된 데이터는 DB에서 영구 제거됩니다.

**예방 방법**:
- 삭제 전 CSV 백업 (정기적으로 전체 목록 백업)
- 삭제 대신 **unsubscribed 상태로 변경** (복구 가능)
- Supabase Dashboard → Database → Backups에서 Point-in-time Recovery (24시간 이내)

### Q3: pending 상태가 오래 유지되면 어떻게 하나요?
**A**: pending 상태가 7일 이상 유지되면 다음 조치를 취하세요:
- 이메일 발송 로그 확인
- 인증 이메일 재발송
- 전화/채팅으로 본인 확인 후 수동 승인 (Dropdown 메뉴 → "확인 완료로 변경")

### Q4: 구독자에게 이메일을 보낼 수 있나요?
**A**: 현재 AdminNewsletter 페이지에서는 **이메일 발송 기능이 없습니다**. 이메일 발송은 별도의 이메일 마케팅 툴을 사용하세요:
- **SendGrid**: 월 100건 무료, API 통합
- **Mailchimp**: 월 500건 무료, UI 간편
- **ConvertKit**: 크리에이터 최적화

### Q5: CSV로 내보내기 기능이 있나요?
**A**: 네, **CSV 내보내기 기능이 있습니다**. 자세한 내용은 [6. CSV 내보내기](#6-csv-내보내기) 섹션을 참고하세요.

### Q6: 구독자 통계는 얼마나 자주 업데이트되나요?
**A**: 구독자 통계는 **실시간으로 업데이트**됩니다 (React Query 캐시: 60초).

**업데이트 시점**:
- 구독자 상태 변경 즉시
- 구독자 삭제 즉시
- 페이지 새로고침 시
- 60초마다 자동 갱신 (React Query staleTime)

### Q7: subscribed_via 값의 의미는 무엇인가요?
**A**: `subscribed_via`는 구독자가 어디서 구독했는지 추적하는 필드입니다. 자세한 내용은 [구독 경로 (subscribed_via)](#구독-경로-subscribed_via) 섹션을 참고하세요.

### Q8: 구독자 개인정보는 어떻게 보호되나요?
**A**: 구독자 개인정보는 다음과 같이 보호됩니다:
- **RLS (Row Level Security)**: 관리자만 접근 가능
- **HTTPS**: 모든 데이터 전송 암호화
- **JWT 토큰**: 인증 토큰 기반 접근 제어
- **SECURITY INVOKER**: 함수 실행 시 RLS 정책 적용

### Q9: 이메일 주소가 중복될 수 있나요?
**A**: 아니요, **이메일 주소는 고유합니다** (UNIQUE 제약조건). 동일 이메일로 재구독 시도 시 기존 레코드 업데이트됩니다.

### Q10: 구독자 수가 갑자기 줄었어요, 왜 그런가요?
**A**: 구독자 수 감소 원인은 다음과 같습니다:
- 사용자가 구독 취소 (unsubscribed)
- 관리자가 구독자 삭제 (GDPR)
- DB 마이그레이션 오류 (Supabase Dashboard 확인)

## 주의사항

### 1. 권한 관리
- **Admin 권한 필수**: AdminNewsletter 페이지는 일반 사용자 접근 불가
- **RLS 정책**: Supabase Row Level Security로 보호
- **비로그인 시**: `/login` 페이지로 리다이렉트

### 2. GDPR 준수
- **삭제는 영구적**: 복구 불가능, 신중히 결정
- **법적 근거**: GDPR Article 17 (Right to Erasure)
- **삭제 요청 문서화**: 사용자 요청 이메일, 티켓 번호 보관

### 3. 이메일 발송 주의
- **unsubscribed 상태**: 이메일 발송 금지 (법적 의무)
- **confirmed 상태만 발송**: pending 상태에는 인증 이메일만 발송

### 4. 데이터 백업
- **정기 백업**: 주 1회 전체 CSV 내보내기 권장
- **삭제 전 백업**: 중요 구독자 삭제 전 CSV 저장

### 5. 성능 고려사항
- **페이지네이션**: 50개씩 표시 (고정)
- **통계 캐시**: 60초 (staleTime)

### 6. 보안
- **SQL Injection 방지**: Prepared Statements (Supabase 자동)
- **XSS 방지**: React 자동 이스케이프
- **CSRF 방지**: JWT 토큰 인증
