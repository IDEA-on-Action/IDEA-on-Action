# 사이트 재구조화 요구사항

> 사용자 관점에서 회사의 정체성과 서비스를 명확하게 전달하는 사이트 구조

**작성일**: 2025-11-23
**버전**: 1.0.0
**상태**: Draft

---

## 1. 배경 및 문제점

### 현재 메뉴 구조
```
홈 | 회사소개 | 서비스 | 로드맵 | 포트폴리오 | 실험실 | 협업하기
```

### 문제점

1. **메뉴 역할 불명확**: 7개 메뉴가 각각 무엇을 차별화하는지 모호함
2. **정체성 전달 실패**: "이 회사가 무슨 생각으로 무슨 일을 하는지" 한눈에 파악 어려움
3. **콘텐츠 분절**: 로드맵, 포트폴리오, 실험실이 별개로 존재하여 통합된 스토리 부재
4. **실시간성 부재**: 프로젝트 진행상황이 실시간 반영되지 않음
5. **소통 채널 부재**: 블로그가 숨김 처리되어 있어 회사의 생각을 공유할 창구 없음

## 2. 사용자 스토리

### 2.1 잠재 고객
> "이 회사가 뭘 하는 회사인지 3초 안에 알고 싶다"
> "어떤 서비스를 얼마에 받을 수 있는지 바로 보고 싶다"
> "실제로 뭘 만들고 있는지, 잘 하는 회사인지 확인하고 싶다"

### 2.2 관심있는 방문자
> "이 회사의 철학과 생각이 궁금하다"
> "현재 진행중인 프로젝트가 어디까지 왔는지 알고 싶다"
> "정기적으로 소식을 받아보고 싶다"

### 2.3 협업 희망자
> "어떻게 연락하면 되는지 바로 찾고 싶다"
> "채용 중인지, 함께할 수 있는 기회가 있는지 알고 싶다"

## 3. 요구사항

### 3.1 메뉴 구조 단순화

**기존 7개 → 신규 5개**

| 순서 | 메뉴 | 역할 정의 | URL |
|------|------|-----------|-----|
| 1 | 홈 | 첫인상 + 핵심 가치 전달 | `/` |
| 2 | 서비스 | 우리가 파는 것 (가격 포함) | `/services` |
| 3 | 프로젝트 | 우리가 만드는 것 (실시간 현황) | `/projects` |
| 4 | 이야기 | 우리가 나누는 것 (블로그/뉴스레터) | `/stories` |
| 5 | 함께하기 | 연결 창구 (문의/채용/커뮤니티) | `/connect` |

### 3.2 각 메뉴 세부 요구사항

#### 3.2.1 홈 (`/`)
- 3초 안에 회사 정체성 전달
- 슬로건: "생각을 멈추지 않고, 행동으로 옮깁니다"
- 섹션 구성:
  - Hero: 핵심 메시지 + CTA
  - What We Do: 서비스 요약 (Development + Minu)
  - What We're Building: 진행중 프로젝트 Top 3 (진척률 포함)
  - Latest Updates: 최근 소식 3개
  - Newsletter CTA: 구독 유도

#### 3.2.2 서비스 (`/services`)
- 명확한 가격과 가치 제안
- 2개 카테고리 명확 구분:
  - Development Services (프로젝트 기반, 일회성)
  - Minu Platform (SaaS 구독, 월간)
- 각 서비스별 상세 페이지 필수
- 가격 정보 명시 (토스페이먼츠 심사 요건)

#### 3.2.3 프로젝트 (`/projects`)
- 기존 포트폴리오 + 로드맵 + 실험실 통합
- 상태별 필터: 진행중 / 출시됨 / 실험중 / 예정
- 각 프로젝트별 정보:
  - 진척률 (0-100%)
  - 상태 배지
  - GitHub 연동 정보 (커밋, 이슈)
  - 기여자, 기술 스택, 타임라인
- 로드맵: 분기별 계획 (기존 기능 유지)

#### 3.2.4 이야기 (`/stories`)
- 회사의 생각과 진행상황 공유
- 하위 섹션:
  - 블로그: WordPress 연동 (숨김 해제)
  - 뉴스레터: 발송된 뉴스레터 아카이브
  - Changelog: 서비스/프로젝트 릴리즈 노트
  - 공지사항: 기존 기능 이전
- 프로젝트 진행상황 자동 포스팅 연동

#### 3.2.5 함께하기 (`/connect`)
- 모든 연결 창구 통합
- 하위 섹션:
  - 프로젝트 문의: 협업 제안 폼
  - 채용: 바운티 연결, 채용 공고
  - 커뮤니티: 기존 커뮤니티 페이지 이전

### 3.3 실시간 동기화 기능

#### A. 프로젝트 진척률 자동 동기화
- GitHub API 연동
- 커밋/이슈/PR 카운트 실시간 반영
- 마일스톤 기반 진척률 계산
- Supabase 실시간 구독

#### B. 릴리즈 알림 시스템
- GitHub Release 감지
- Supabase 저장 및 Changelog 자동 업데이트
- 자동 블로그 포스트 생성 (선택적)
- 뉴스레터 포함 (선택적)

#### C. 뉴스레터 확장
- 웹 아카이브 페이지
- 주간 요약 자동 생성 (프로젝트 진행상황 포함)
- 구독자 세그먼트별 발송 (선택적)

### 3.4 기존 페이지 처리

| 기존 페이지 | 처리 방법 |
|-------------|-----------|
| `/about` | 홈에 섹션으로 통합, 301 리디렉션 |
| `/roadmap` | `/projects?tab=roadmap` 리디렉션 |
| `/portfolio` | `/projects` 리디렉션 |
| `/lab` | `/projects?tab=lab` 리디렉션 |
| `/work-with-us` | `/connect` 리디렉션 |
| `/blog` | `/stories/blog` 리디렉션 |
| `/notices` | `/stories/notices` 리디렉션 |
| `/community` | `/connect/community` 리디렉션 |

## 4. 비기능 요구사항

### 4.1 성능
- Lighthouse 점수 90+ 유지
- 초기 로딩 3초 이내
- Core Web Vitals 통과

### 4.2 접근성
- WCAG 2.1 AA 준수
- 키보드 네비게이션 지원
- 스크린 리더 호환

### 4.3 반응형
- 모바일 퍼스트 디자인
- 태블릿/데스크톱 최적화

### 4.4 SEO
- 메타 태그 최적화
- 구조화된 데이터 (JSON-LD)
- sitemap.xml 업데이트

- 다국어 지원 확장 (별도 Phase)
- AI 챗봇 고도화 (별도 Phase)
- 관리자 페이지 재구조화 (현재 구조 유지)

## 6. 성공 기준

1. 방문자가 3초 안에 회사 정체성을 파악할 수 있음
2. 모든 서비스의 가격이 명확하게 노출됨
3. 프로젝트 진행상황이 실시간으로 확인 가능함
4. 뉴스레터/블로그를 통해 정기적인 소통이 이루어짐
5. 메뉴 클릭 시 해당 페이지의 역할이 명확함
