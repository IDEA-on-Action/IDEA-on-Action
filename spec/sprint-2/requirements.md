## 🎯 Sprint 개요

### 목표
Sprint 1에서 구축한 정적 페이지 구조를 **실시간 업데이트 가능한 동적 시스템**으로 전환하고, **커뮤니티 참여 기능**을 추가합니다.

### 핵심 가치
1. **투명성**: 프로젝트 진행 상황을 실시간으로 공유
2. **참여**: 방문자가 댓글/문의를 통해 직접 참여 가능
3. **연결**: 뉴스레터로 지속적인 관계 유지
4. **효율성**: 관리자가 쉽게 콘텐츠 업데이트 가능

### Sprint 2에서 달성할 것
- ✅ 프로젝트/로드맵/활동 로그를 **실시간 업데이트** 가능하게
- ✅ 커뮤니티 페이지에서 **토론과 피드백** 주고받기
- ✅ 협업 희망자가 **직접 문의** 가능
- ✅ 관심 있는 방문자가 **뉴스레터 구독** 가능

## 👤 사용자 스토리

### 1. 방문자 (Visitor)

#### US-2.1: 최신 프로젝트 현황 확인
**As a** 방문자
**I want to** Portfolio 페이지에서 최신 프로젝트 현황을 확인하고
**So that** IDEA on Action이 어떤 일을 하고 있는지 이해할 수 있다

**예시**:
- "지난주에 봤을 때 'AI 챗봇' 프로젝트가 'In Progress'였는데, 오늘 보니 'Launched'로 바뀌었어요!"
- "새로운 프로젝트 'Vibe Coding Agent'가 추가됐네요. 어떤 프로젝트인지 궁금해요."

**가치**: 실시간 업데이트로 회사의 **활발한 활동**을 느낄 수 있음

---

#### US-2.2: 로드맵 진행률 추적
**As a** 방문자
**I want to** Roadmap 페이지에서 분기별 목표와 진행률을 보고
**So that** 회사의 방향성과 달성도를 파악할 수 있다

**예시**:
- "2025 Q4 목표: 커뮤니티 기능 구축 - 진행률 80%"
- "책임자가 'Sinclair Seo'라고 명시되어 있어서 신뢰가 가네요."

**가치**: 투명한 목표 공유로 **신뢰도 향상**

---

#### US-2.3: 최근 활동 로그 확인
**As a** 방문자
**I want to** Now 페이지에서 최근 의사결정/학습/릴리스 로그를 보고
**So that** 회사가 실제로 활발히 움직이고 있음을 확인할 수 있다

**예시**:
- "2025-11-14: Weekly Recap 자동화 완료 (Decision)"
- "2025-11-13: PostgreSQL RLS 정책 디버깅 경험 (Learning)"
- "2025-11-12: OG Image 생성 스크립트 릴리스 (Release)"

**가치**: 실시간 활동 로그로 **생동감** 전달

---

### 2. 커뮤니티 멤버 (Community Member)

#### US-2.4: 커뮤니티 페이지에서 토론 참여
**As a** 커뮤니티 멤버
**I want to** Community 페이지에서 다른 사람들과 토론하고
**So that** 아이디어를 공유하고 피드백을 받을 수 있다

**예시**:
- "AI 챗봇 프로젝트에 대해 궁금한 게 있어요. 어떤 모델을 사용하셨나요?"
- "저도 비슷한 프로젝트를 해봤는데, 이런 방식은 어떨까요?"

**가치**: **쌍방향 소통**으로 커뮤니티 활성화

---

#### US-2.5: 블로그 포스트에 댓글 남기기
**As a** 커뮤니티 멤버
**I want to** Blog 포스트 하단에 댓글을 남기고
**So that** 글쓴이와 다른 독자들과 의견을 나눌 수 있다

**예시**:
- "Weekly Recap을 GitHub Actions로 자동화한 아이디어 좋네요!"
- "RLS 정책 디버깅 과정 공유 감사합니다. 저도 같은 문제로 고생했었어요."

**가치**: 블로그를 **대화의 장**으로 전환

---

### 3. 협업 희망자 (Potential Partner)

#### US-2.6: Work with Us 폼으로 문의
**As a** 협업 희망자
**I want to** Work with Us 페이지에서 프로젝트 의뢰 폼을 작성하고
**So that** IDEA on Action에 직접 연락할 수 있다

**예시**:
- 패키지 선택: "AI 컨설팅 (₩3,000,000)"
- 프로젝트 브리프: "AI 챗봇을 우리 고객센터에 도입하고 싶습니다. 상담 가능할까요?"
- 제출 → "문의가 접수되었습니다. 영업일 기준 2일 내 답변드리겠습니다."

**가치**: **즉시 연락** 가능한 창구 제공

---

### 4. 관심 구독자 (Subscriber)

#### US-2.7: 뉴스레터 구독
**As a** 관심 구독자
**I want to** 이메일 주소만 입력해서 뉴스레터를 구독하고
**So that** 주간 요약과 새 프로젝트 소식을 받을 수 있다

**예시**:
- Footer에서 이메일 입력: "yourname@example.com"
- 제출 → "뉴스레터 구독 신청 완료! 매주 월요일 아침 발송됩니다."
- 다음 월요일: "📬 IDEA on Action Weekly Recap #1" 이메일 수신

**가치**: **지속적인 관계** 유지

## 🎨 기능 요구사항

### FR-2.1: 동적 콘텐츠 관리 (Dynamic Content)

#### FR-2.1.1: 프로젝트 데이터
- **요구사항**: Portfolio 페이지의 프로젝트 정보를 데이터베이스에서 불러오기
- **데이터 항목**:
  - 제목, 요약, 상태 (Backlog/In Progress/Validate/Launched)
  - 태그 (예: "AI", "Automation", "Community")
  - 메트릭 (예: "사용자 100명", "만족도 95%")
  - 관련 링크 (GitHub, Demo, Docs)
- **업데이트**: 관리자가 데이터베이스에서 직접 수정 시 즉시 반영
- **필터링**: 상태별, 태그별 필터 기능

**사용자 가치**: 항상 **최신 정보** 확인 가능

---

#### FR-2.1.2: 로드맵 데이터
- **요구사항**: Roadmap 페이지의 분기별 목표를 데이터베이스에서 불러오기
- **데이터 항목**:
  - 분기 (예: "2025 Q4")
  - 목표 (예: "커뮤니티 기능 구축")
  - 진행률 (0~100%)
  - 책임자 (예: "Sinclair Seo")
  - 관련 프로젝트 ID 목록
- **시각화**: 진행률 링 차트, 책임자 프로필
- **업데이트**: 주간 단위로 진행률 갱신

**사용자 가치**: **투명한 목표 추적**

---

#### FR-2.1.3: 활동 로그 데이터
- **요구사항**: Now 페이지의 최근 활동을 데이터베이스에서 불러오기
- **데이터 항목**:
  - 타입 (Decision/Learning/Release)
  - 내용 (텍스트)
  - 관련 프로젝트 ID
  - 작성일시
- **정렬**: 최신순
- **표시**: 최근 10개

**사용자 가치**: **실시간 활동 현황** 파악

---

#### FR-2.1.4: 바운티 데이터
- **요구사항**: Lab 페이지의 오픈 바운티 정보를 데이터베이스에서 불러오기
- **데이터 항목**:
  - 제목, 필요 스킬, 보상 금액, 마감일
  - 상태 (Open/Assigned/Done)
  - 지원자 목록
- **필터링**: 스킬별, 상태별
- **신청**: 로그인 후 "지원하기" 버튼

**사용자 가치**: **참여 기회** 제공

---

### FR-2.2: 커뮤니티 기능 (Community Features)

#### FR-2.2.1: Community 페이지 댓글 시스템
- **요구사항**: Community 페이지에 실시간 댓글 기능 추가
- **기능**:
  - GitHub 계정으로 로그인
  - 댓글 작성/수정/삭제
  - 댓글에 답글 달기
  - 좋아요/싫어요 (Upvote/Downvote)
  - 마크다운 문법 지원
- **모더레이션**: 관리자가 부적절한 댓글 삭제 가능

**사용자 가치**: **자유로운 토론** 공간

---

#### FR-2.2.2: Blog 포스트 댓글 시스템
- **요구사항**: 각 블로그 포스트 하단에 댓글 섹션 추가
- **기능**: FR-2.2.1과 동일
- **표시**: 댓글 개수 뱃지 (예: "💬 5 comments")

**사용자 가치**: 블로그 **피드백 수집**

---

### FR-2.3: 협업 문의 (Work with Us)

#### FR-2.3.1: 문의 폼
- **요구사항**: Work with Us 페이지에 프로젝트 의뢰 폼 추가
- **입력 필드**:
  - 이름 (필수)
  - 이메일 (필수, 이메일 형식 검증)
  - 회사명 (선택)
  - 패키지 선택 (AI 컨설팅/프로덕트 개발/워크플로우 자동화)
  - 예산 범위 (선택)
  - 프로젝트 브리프 (필수, 최소 50자)
- **제출**:
  - 데이터베이스 저장
  - 관리자 이메일 알림 발송
  - 사용자에게 확인 토스트 메시지

**사용자 가치**: **간편한 문의** 절차

---

#### FR-2.3.2: 알림 통합
- **요구사항**: 폼 제출 시 실시간 알림
- **알림 수단**:
  - 이메일 (관리자 메일)
  - 또는 Slack Webhook
  - 또는 캘린더 연동 (Cal.com / Google Calendar)
- **알림 내용**:
  - 제출 시각
  - 문의자 정보 (이름, 이메일, 회사)
  - 선택 패키지 & 예산
  - 프로젝트 브리프 전문

**사용자 가치**: **빠른 응답** 기대

---

### FR-2.4: 뉴스레터 (Newsletter)

#### FR-2.4.1: 구독 위젯
- **요구사항**: Footer에 뉴스레터 구독 폼 추가
- **입력 필드**:
  - 이메일 (필수, 이메일 형식 검증)
- **제출**:
  - 데이터베이스 저장
  - 중복 확인 (이미 구독 중이면 "이미 구독 중입니다" 메시지)
  - 확인 토스트: "뉴스레터 구독 신청 완료!"

**사용자 가치**: **주간 소식** 수신

---

#### FR-2.4.2: 구독자 관리
- **요구사항**: 관리자가 구독자 목록 확인 및 발송 관리
- **기능**:
  - 구독자 목록 조회 (이메일, 가입일)
  - 구독 상태 (활성/비활성)
  - CSV 내보내기 (이메일 리스트)
- **발송**: 외부 이메일 서비스 연동 (Resend / Beehiiv)

**사용자 가치**: 관리자 **효율적 운영**

## 🛤️ 사용자 여정

### Journey 1: 프로젝트 진행 상황 확인

```
방문자 방문 (www.ideaonaction.ai)
  ↓
Home 페이지 → Portfolio Highlight 섹션 (3개 프로젝트 미리보기)
  ↓
"더 보기" 클릭 → Portfolio 페이지 이동
  ↓
프로젝트 목록 (6개) → 상태별 필터 (In Progress만 보기)
  ↓
프로젝트 카드 클릭 → 상세 페이지
  ↓
"GitHub 보기" 링크 → 실제 코드 확인
  ↓
만족 → 뒤로 가기 → Footer에서 뉴스레터 구독
```

**예상 소요 시간**: 3~5분
**성공 기준**: 뉴스레터 구독 완료

### Journey 2: 커뮤니티 토론 참여

```
커뮤니티 멤버 방문 (www.ideaonaction.ai/community)
```

**성공 기준**: 댓글 1개 이상 작성

### Journey 3: 프로젝트 의뢰 문의

```
협업 희망자 방문 (www.ideaonaction.ai/work-with-us)
  ↓
Work with Us 페이지 → 패키지 소개 확인
  ↓
"AI 컨설팅" 패키지 선택

  ↓
"문의하기" 클릭 → 폼 검증 (이메일 형식, 브리프 길이)
  ↓
제출 성공 → 토스트 메시지: "문의 접수 완료! 영업일 2일 내 답변드립니다."
  ↓
관리자 이메일 알림 발송
  ↓
2일 후 → 담당자 이메일 회신 수신
```

**예상 소요 시간**: 5~7분

## 📊 성공 지표

### 정량적 지표 (Quantitative)

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| **동적 데이터 연결** | 4개 테이블 (projects, roadmap, logs, bounties) | 데이터베이스 조회 성공 |
| **커뮤니티 참여** | 댓글 5개 이상 | Community/Blog 댓글 수 집계 |
| **문의 접수** | 문의 2건 이상 | Work with Us 폼 제출 수 |
| **뉴스레터 구독** | 구독자 10명 이상 | Newsletter 테이블 레코드 수 |
| **페이지 로딩 속도** | 3초 이내 | Lighthouse Performance 90+ |

### 정성적 지표 (Qualitative)

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| **데이터 신뢰도** | 방문자가 "항상 최신 정보"라고 느낌 | 사용자 피드백 (댓글) |
| **커뮤니티 활성도** | 댓글이 건설적이고 활발함 | 댓글 평균 길이 > 50자 |

| **뉴스레터 만족도** | 구독 후 1주일 내 이탈 < 10% | 구독 취소율 |
