# 프롬프트 템플릿 관리 인수 조건 명세서

> BDD (Behavior-Driven Development) 형식의 검증 가능한 인수 조건

**작성일**: 2025-11-25
**버전**: 1.0.0
**상태**: Active
**관련 문서**: [requirements.md](./requirements.md)

---

## 1. 개요

이 문서는 프롬프트 템플릿 관리 기능의 **검증 가능한 인수 조건**을 정의합니다.
각 조건은 **Given-When-Then** 형식으로 작성되어 명확한 테스트 시나리오를 제공합니다.

---

## 2. 기능적 인수 조건

### AC-PT-01: 템플릿 생성 성공

**사용자 스토리**: US-PT-01 (템플릿 저장)

**시나리오 1: 기본 정보만으로 생성**
```gherkin
Feature: 프롬프트 템플릿 생성
  As a Minu 서비스 사용자
  I want to AI 프롬프트를 템플릿으로 저장
  So that 다음에 재사용할 수 있습니다

Scenario: 기본 정보만으로 템플릿 생성
  Given 사용자가 Minu Frame에서 RFP를 생성했고
  And AI 응답에 만족했을 때
  When "템플릿으로 저장" 버튼을 클릭하고
  And 이름에 "정부 SI RFP 기본"을 입력하고
  And 카테고리로 "rfp"를 선택하고
  And "저장" 버튼을 클릭하면
  Then 템플릿이 성공적으로 생성되고
  And "템플릿이 저장되었습니다" 토스트가 표시되고
  And 내 템플릿 목록에 새 템플릿이 보입니다

Acceptance Criteria:
  ✓ 필수 필드 (name, user_prompt_template) 검증
  ✓ user_id는 현재 로그인 사용자로 자동 설정
  ✓ 기본값 적용: is_public=false, status=active, usage_count=0
  ✓ created_at, updated_at 자동 생성
  ✓ 성공 시 201 Created 응답
```

**시나리오 2: 변수가 포함된 템플릿 생성**
```gherkin
Scenario: 변수가 포함된 템플릿 생성
  Given 사용자가 프롬프트에 "{{project_name}}"와 "{{budget}}"를 포함했을 때
  When 템플릿 저장 다이얼로그를 열면
  Then "2개의 변수를 감지했습니다" 메시지가 표시되고
  And 변수 목록에 project_name, budget이 표시되고
  When 각 변수에 라벨을 입력하고 저장하면
  Then variables 필드에 JSON 배열로 저장됩니다
```

**검증 포인트**:
- [ ] 이름 최대 100자 제한
- [ ] user_prompt_template 최대 10KB 제한
- [ ] 변수 자동 감지 정규식: `/\{\{([a-zA-Z0-9_]+)\}\}/g`
- [ ] RLS 정책으로 다른 사용자의 템플릿 조회 불가

### AC-PT-03: 템플릿 목록 조회

```gherkin

  Given 팀원 A가 3개의 템플릿을 팀과 공유했을 때

  And 수정, 삭제 버튼은 표시되지 않습니다
```

**시나리오 3: 시스템 템플릿 조회**
```gherkin
Scenario: IDEA on Action 제공 기본 템플릿 조회
  Given 시스템에 10개의 기본 템플릿이 등록되어 있을 때
  When "시스템 템플릿" 탭을 열면
  Then 10개의 템플릿이 카테고리별로 그룹화되어 표시되고
  And 각 템플릿에는 "사용하기", "복사하기" 버튼이 제공됩니다
```

**검증 포인트**:
- [ ] 페이지네이션: 20개씩
- [ ] 무한 스크롤 또는 페이지 버튼
- [ ] 로딩 상태: 스켈레톤 UI
- [ ] 빈 상태: "아직 템플릿이 없습니다" 메시지
- [ ] 응답 시간 P95 < 500ms

**시나리오 2: 다중 변수 감지**
```gherkin
Scenario: 여러 변수가 포함된 템플릿
  Given 프롬프트에 "{{project_name}}, {{budget}}원, {{duration}}개월"이 있을 때
  When 저장 버튼을 클릭하면
```

- [ ] 정규식: `/\{\{([a-zA-Z0-9_]+)\}\}/g`

- [ ] 중복 제거 (Set 사용)
- [ ] 변수가 없으면 variables 필드는 null 또는 빈 배열

### AC-PT-11: 성능 요구사항

**시나리오 1: 목록 조회 성능**
```gherkin
Scenario: 템플릿 100개 조회 성능
  Given 사용자가 100개의 템플릿을 가지고 있을 때
  When "내 템플릿" 탭을 열면
  Then 첫 20개가 500ms 이내에 표시되고
  And 스크롤 시 추가 로딩도 500ms 이내입니다
```

**시나리오 2: 변수 렌더링 성능**
```gherkin
Scenario: 변수 10개 렌더링 성능
  Given 템플릿에 10개의 변수가 있을 때
  When 모든 변수에 값을 입력하면
  Then 렌더링 결과가 100ms 이내에 표시됩니다
```

**검증 포인트**:
- [ ] 목록 조회 P95 < 500ms
- [ ] 변수 렌더링 P95 < 100ms
- [ ] 페이지네이션으로 무거운 로딩 방지
- [ ] React Query로 캐싱 (5분)

---

### AC-PT-12: 보안 요구사항

**시나리오 1: 본인 템플릿만 수정 가능**
```gherkin
Scenario: 다른 사용자의 템플릿 수정 시도
  Given 사용자 A가 사용자 B의 템플릿 ID를 알고 있을 때
  When API를 통해 직접 UPDATE 요청을 보내면
  Then 403 Forbidden 응답이 반환되고
  And "권한이 없습니다" 에러 메시지가 표시됩니다
```

**시나리오 2: SQL Injection 방어**
```gherkin
Scenario: 악의적인 입력 시도
  Given 사용자가 템플릿 이름에 "'; DROP TABLE prompt_templates; --"를 입력했을 때
  When 저장 버튼을 클릭하면
  Then 입력값이 이스케이프 처리되어 안전하게 저장되고
  And 데이터베이스에 피해가 없습니다
```

**검증 포인트**:
- [ ] RLS 정책 적용 (user_id 체크)
- [ ] 파라미터 바인딩 사용 (SQL Injection 방어)
- [ ] XSS 방어 (렌더링 시 이스케이프)
- [ ] Rate Limiting (분당 10회)

---

### AC-PT-13: 접근성 요구사항

**시나리오 1: 키보드 네비게이션**
```gherkin
Scenario: 키보드만으로 템플릿 생성
  Given 사용자가 마우스를 사용할 수 없을 때
  When Tab 키로 "새 템플릿" 버튼으로 이동하고
  And Enter 키로 다이얼로그를 열고
  And Tab 키로 입력 필드를 순회하고
  And Enter 키로 저장하면
  Then 템플릿이 성공적으로 생성됩니다
```

**시나리오 2: 스크린 리더 지원**
```gherkin
Scenario: 스크린 리더로 템플릿 목록 탐색
  Given 시각 장애인 사용자가 스크린 리더를 사용할 때
  When 템플릿 목록 페이지를 열면
  Then 각 템플릿의 이름, 카테고리, 사용 횟수가 음성으로 읽히고
  And 버튼의 역할이 명확히 안내됩니다
```

**검증 포인트**:
- [ ] 모든 인터랙티브 요소에 포커스 스타일
- [ ] ARIA 레이블 (다이얼로그, 토글 버튼)
- [ ] 의미론적 HTML (button, input, form)
- [ ] 키보드 단축키 (Esc로 다이얼로그 닫기)

### AC-PT-14: 사용성 요구사항

**시나리오 1: 에러 메시지 명확성**
```gherkin
Scenario: 네트워크 에러 발생 시
  Given 사용자가 오프라인 상태일 때
  When 템플릿을 저장하려고 하면
  Then "네트워크 연결을 확인해주세요" 메시지가 표시되고
  And 재시도 버튼이 제공됩니다
```

**시나리오 2: 로딩 상태 피드백**
```gherkin
Scenario: 템플릿 목록 로딩 중
  Given 사용자가 템플릿 목록을 요청했을 때
  When 응답이 도착하기 전까지
  Then 스켈레톤 UI가 표시되고
  And 20개의 카드 플레이스홀더가 보입니다
```

**검증 포인트**:
- [ ] 모든 비동기 작업에 로딩 상태
- [ ] 에러는 토스트 또는 인라인 메시지로 표시
- [ ] 성공 액션은 토스트 알림 (3초 자동 닫힘)
- [ ] 빈 상태 메시지 + CTA 버튼

### AC-PT-15: 확장성 요구사항

**시나리오 1: 새 변수 타입 추가**
```gherkin
Scenario: select 타입 변수 지원
  Given 향후 변수 타입에 'select'가 추가될 때
  When 템플릿에 {{ category:select }} 형식을 사용하면
  Then 드롭다운 선택 UI가 제공되고
  And 기존 text, number 타입과 공존합니다
```

**시나리오 2: 새 카테고리 추가**
```gherkin
Scenario: Minu Monitor 서비스 추가 시
  Given 새로운 Minu 서비스가 출시될 때
  When 카테고리에 'monitor'를 추가하면
  Then 기존 코드 수정 없이 작동하고
  And 필터, 정렬이 자동으로 적용됩니다
```

**검증 포인트**:
- [ ] variables 스키마 확장 가능 (JSONB 타입)
- [ ] 카테고리는 Enum이 아닌 text 타입
- [ ] 플러그인 구조로 변수 타입 추가 가능
- [ ] i18n 지원 가능한 데이터 구조
